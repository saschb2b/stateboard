{{- else }}
  kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "stateboard.fullname" . }} 3000:{{ .Values.service.port }}
  open http://localhost:3000
{{- end }}

Auth setup checklist:
  1. In Keycloak, create a confidential client with the redirect URI
     {{ .Values.auth.baseUrl }}/api/auth/callback/keycloak
  2. Copy the client secret into auth.keycloak.clientSecret (or your
     existingSecret as KEYCLOAK_CLIENT_SECRET).
  3. Set auth.secret to a 32-byte random string: openssl rand -base64 32
  4. The first user to sign in becomes the workspace owner. Subsequent
     users land as {{ .Values.auth.defaultRole }} (configurable via auth.defaultRole).

Reminders:
  • Telemetry is off, the app makes no outbound calls except to Keycloak. Airgap-friendly.
  • Back up the Postgres database AND the uploads PVC ({{ include "stateboard.pvcName" . }}).
  • If running replicaCount > 1, set persistence.accessMode: ReadWriteMany.
